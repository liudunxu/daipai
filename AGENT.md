# AGENT.md - Claude Code 行为规范

本文档定义 Claude Code 在本项目中的行为规范。

## 项目信息

- **项目名称**: 极客观察
- **类型**: Next.js 14 多功能工具类网站
- **网址**: https://www.zkwatcher.top
- **详细规范**: 见 [CLAUDE.md](CLAUDE.md)

## 核心规则

### 页面管理
- **新增或删除页面时，必须同步更新 `/nav` 导航页面的链接**
- 保持导航页面包含所有有效页面的入口

### Git 提交
- 每次修改完成后自动提交
- 提交格式: `feat: xxx`, `fix: xxx`, `style: xxx`, `refactor: xxx`, `docs: xxx`, `perf: xxx`
- 提交信息使用中文描述

### 构建验证
- **推送前必须本地构建通过**: `npm run build`
- 如有错误自动修复后再次构建验证

## 任务判定

| 条件 | 处理方式 |
|------|----------|
| 涉及 3+ 文件修改 | 进入 plan 模式 |
| 新增页面/路由/API | 进入 plan 模式 |
| 引入新依赖 | 进入 plan 模式 |
| 样式修改、Bug 修复、文案更新 | 直接执行 |

## 编码准则

### 简洁优先
- 不写 speculative code（推测性代码）
- 不添加未请求的功能
- 不为单次使用的代码创建抽象
- 不添加"灵活性"或"可配置性"除非确实需要

### 精准修改
- 只修改必须修改的代码
- 不"改进"相邻代码、注释或格式
- 匹配现有代码风格
- 更改创建的无用代码要清理

### 假设确认
- 明确陈述假设，不确定的要问
- 多个可行方案时，说明而不选择
- 存在更简单方案时，指出
- 不明确时停止，说出疑问

## 注意事项

1. 外部图片建议使用本地图片或 base64
2. JSON 文件数据是临时的，生产环境使用 Vercel KV
3. 儿童类工具优先使用选择器，减少手动输入
4. API 域名已迁移至 Cloudflare Workers
